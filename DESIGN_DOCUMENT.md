## 1. 系统概述

AI 智能报销系统是一套面向企业的费控管理平台，核心目标是通过 AI 技术自动化处理出差申请、差旅报销、员工报销等业务流程，降低人工录入成本，提升合规性校验效率。

### 1.1 核心功能模块

| 模块 | 路由 | 描述 |
|------|------|------|
| 首页（AI 对话） | `/` | 智能意图识别，语音/文本输入，快捷流程入口 |
| 出差申请 | `/business-trip` | 2步骤流程，智能计算公里数/天数/费用/补贴 |
| 差旅报销 | `/travel-reimbursement` | 4步骤流程，票据上传→AI稽查→信息填充→合规校验 |
| 员工报销 | `/employee-reimbursement` | 4步骤流程，同差旅报销结构，针对日常费用 |
| 影像管理 | `/image-management` | 票据分类展示、搜索、批量操作、票据预览 |
| 字段配置 | `/field-config` | 拖拽式表单设计器，三面板布局 |
| 规则配置 | `/rule-config` | 企业费控规则引擎管理，支持多种规则类型 |

### 1.2 技术架构

```

│   ├── App.tsx                    # 根组件，RouterProvider + Toaster
```

```typescript
function analyzeIntent(text: string): { label, path } | null {
  // 匹配规则（优先级从高到低）：
  // "出差申请" | ("出差" && !"报销") → /business-trip
  // "差旅报销" | "差旅"              → /travel-reimbursement
  // "员工报销"                       → /employee-reimbursement
  // "报销" && !"差旅" && !"员工"    → /employee-reimbursement
  // "影像" | "票据管理" | "发票管理"  → /image-management
  // "字段" | "配置" | "规则"         → /field-config
}
```

**交互流程**：
1. 用户输入文字后，AI 显示"正在识别..."打字动画（1秒延迟）
2. 识别到意图 → 回复确认文本 + 跳转按钮，1.5秒后自动导航
3. 未识别意图 → 回复帮助菜单 + 三个快捷按钮

#### 智能自动计算（useEffect 依赖项）

触发字段：`startDate` / `returnDate` / `departure` / `destination` / `tripType` / `transport`

```typescript
// 出差天数：(returnDate - startDate) + 1，最小值 1
// 往返公里数：城市距离表查找，默认 500km
// 差旅补贴资格：排除"总部售后培训"和"总部销售培训"类型，且 km >= 200
// 差旅补贴金额：天数 × 90 元/天
// 长途交通费：按交通工具×公里数公式计算
//   飞机：单程 × 0.8 + 300
//   高铁：单程 × 0.45
//   火车：单程 × 0.25
//   其他：单程 × 0.3
// 住宿费：(天数-1) × 标准（一线城市330元，其他220元）
// 市内交通费：天数 × 60 + 80
// 合计：交通 + 住宿 + 市内交通 + 补贴
```

| 城市对 | 距离 |

| 上海-广州 | 2430 |
| 上海-深圳 | 2550 |
| 上海-成都 | 3378 |
| 上海-武汉 | 1282 |
| 上海-杭州 | 338 |
| 上海-南京 | 588 |

## 7. 差旅报销

**文件**：`TravelReimbursement.tsx`  
**路由**：`/travel-reimbursement`  
**步骤**：票据上传 → 票据稽查 → 信息填充 → 信息检查（4步）

### 7.1 步骤一：票据上传

- **说明横幅**：黄色提示框，说明支持差旅票据类型
- **拖拽上传区**：`border-2 border-dashed`，支持 drag over 颜色变化
- **快捷上传按钮**：拍照上传 / 上传附件（2列按钮）
  - 拍照上传：设置 `hasUploadedCamera = true`（影响稽查结果）
- **已上传列表**：显示文件名、类型图标（PDF红色 / 图片蓝色）、删除按钮
- **默认预置文件**：交通票.png、住宿发票.pdf

### 7.2 步骤二：票据稽查

状态机：`"checking" | "pass" | "fail"`

| 状态 | 触发条件 | 视觉 |
|------|---------|------|
| checking | 进入步骤时 | 旋转动画图标 + "正在深度分析..." |
| pass | 使用"上传附件"（2.5秒后） | 绿色成功框 |
| fail | 使用"拍照上传"（2.5秒后） | 红色错误框 + 错误列表 + 重新上传按钮 |

**稽查失败错误信息**：
1. 票据金额与实际报销金额不符
2. 1号发票疑似造假，无法查到对应发票
3. 2号发票识别出标题异常
